  Project at a glance

  Palindrome — Expo / React Native game (iOS, Android, Web) shipping as
  com.gammagames.palindrome*. Stack: Expo Router 6, React Native 0.81 + RN-Web,
  Reanimated 4, Supabase 2 (auth + Postgres + realtime), Google/Apple sign-in,
  expo-audio, AsyncStorage. Theming and settings via two contexts; auth via
  useAuth + authService singleton.

  Layout

  - app/ — Expo-Router routes, _layout.tsx wires ThemeProvider →
  SettingsProvider → splash → gated routing (signed-out users go to
  (tabs)/index/signup, signed-in to (tabs)/main).
  - app/(tabs)/ — every screen has a .tsx + .web.tsx twin (native vs web).
  Largest by far: gamelayout.tsx (2k LOC) / gamelayout.web.tsx (3k LOC), plus
  main.*, profile.*, friends.*, multiplayer.*, signup.*, leaderboard.*,
  notifications.*, addfriend.*, matchwaiting.*, matchresult.*, turngame.* (web
  carries the full impl, native is a 30-line stub).
  - app/auth/ — callback.tsx (OAuth redirect) and verified.tsx (email
  confirmation).
  - lib/ — pure logic & service layer:
    - gameEngine.ts — board init, move validation, palindrome scoring (5×5
  colors, bulldogs +10, deterministic seeded RNG).
    - seededRandom.ts — used by both engine and matches so seeds reproduce.
    - gameColors.ts — default gradients + hex/HSL conversion for color
  customization.
    - matchmaking.ts — Race mode (quick match via claim_quick_match RPC, invite
  codes, realtime subscriptions, score submit, rematch).
    - turnMatchmaking.ts — Turn mode (server-authoritative via claim_turn_match,
   init_turn_board, submit_turn_move, forfeit_turn_match RPCs; turn_match_states
   + turn_match_moves tables).
    - friends.ts — friend requests, challenges (race or turn), head-to-head
  stats, user search.
    - notifications.ts — friend_request | challenge | app_update.
    - singlePlayer.ts — saves runs to single_player_runs, reads from
  single_player_best view + joins profiles.
  - context/SettingsContext.tsx — sound, haptics, color-blind mode
  (symbols/emojis/cards/letters), interaction mode (drag/pick), custom block
  gradients — all persisted in AsyncStorage.
  - context/ThemeContext.tsx — light/dark with hard-coded color tokens.
  - hooks/ — useAuth, use-sound, color-scheme + theme-color helpers.
  - authService.ts — Supabase email/password, Google native + web OAuth, Apple
  native + OAuth, profile ensure/upload to avatars bucket, profile cache in
  AsyncStorage.
  - supabase.ts — lazy client with platform-appropriate storage adapter
  (AsyncStorage native, localStorage web, in-memory SSR fallback).
  - supabase/migrations/ — 16 migrations: profiles → multiplayer tables → RLS
  fixes → rematches → atomic quick match → friends/challenges → notifications →
  single-player scores → turn-mode RLS/RPCs.
